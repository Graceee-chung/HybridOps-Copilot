# Hardware Request and Provisioning

## Issue Pattern
Users need new hardware (laptops, monitors, peripherals), replacements for broken devices, or upgrades for performance issues.

## Common Symptoms
- User requests new laptop or monitor
- Current hardware is broken or not functioning
- User needs upgrade for new role or project
- Hardware is outdated and slow
- Peripherals needed for accessibility
- Mobile device needed for on-site visits

## Hardware Types and Lead Times
- **Laptops:** 5-10 business days after approval
- **Monitors:** 2-3 business days
- **Peripherals (mouse, keyboard, dock):** 1-2 business days
- **Mobile devices:** 5-7 business days
- **Special equipment:** 10-20+ business days

## Troubleshooting / Verification
1. Verify current hardware cannot be repaired (contact IT Support for repair assessment)
2. Confirm user's department has hardware budget available
3. Check if hardware was recently issued (may not qualify for replacement)
4. Document business justification for upgrade requests
5. Capture intended use case (development, data analysis, general office work)
6. Verify user meets minimum security requirements for new hardware
7. Confirm user training on hardware management policy

## Provisioning Steps
1. Submit hardware request through IT Service Portal or this ticketing system
2. Include hardware model preference or IT's recommended spec
3. Include business justification (required for non-replacement requests)
4. Approval from manager (usually automatic for replacements)
5. IT procurement orders hardware
6. Hardware arrives at IT warehouse
7. IT configures hardware with required software and security tools
8. User notified for pickup or shipped directly to home office
9. User confirms hardware receipt and signs off

## Expected Outcomes
- Hardware arrives within specified lead time
- Hardware is pre-configured with required software and security settings
- User receives provisioning instructions and setup guide
- Hardware is registered in asset management system

## Escalation Criteria
- User needs emergency hardware (same day)
- Existing hardware failure impacting critical work
- Budget override needed for expensive requests
- Special equipment requiring custom setup

## Related Issues
- Hardware repair requests
- Software licensing for new hardware
- Network connectivity for new devices
- VPN setup on new hardware

## Escalation Team
IT Assets and Procurement team

## Priority Guidance
- **High:** Hardware failure, user cannot work, emergency replacement
- **Medium:** Upgrade for new role, hardware underperforming
- **Low:** Preference upgrade, can wait for next order batch
