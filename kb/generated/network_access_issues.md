# Network Access Issues

## Issue Pattern
Common issue: Users are unable to access network resources due to incorrect or expired passwords, hardware issues, or configuration problems.

## Common Symptoms
* Password reset requests are not being processed correctly
* External users cannot access company resources due to password expiration
* Network connections are failing or timing out
* Terminal connections are being refused
* Badge access is restricted

## Possible Causes
* Incorrect or expired passwords
* Hardware issues, such as wireless connectivity problems
* Configuration problems, including network settings and user permissions
* Software issues, including application compatibility and database connections

## Troubleshooting Steps

1. Verify the password expiration date for external users and ensure that it has not been reset.
2. Check the network configuration and ensure that all necessary settings are in place.
3. Restart the device or machine to resolve any temporary connectivity issues.
4. Disable and re-enable device credentials to resolve connection problems.
5. Run a troubleshooting script to identify potential software issues.

## Escalation Team
The IT Service Desk team, specifically the Access and Hardware teams, are best suited for resolving network access issues.

## Priority Guidance

* Low priority: If the issue is minor and can be resolved with minimal assistance, such as checking password expiration dates.
* Medium priority: If the issue requires moderate assistance, such as troubleshooting network configuration or disabling device credentials.
* High priority: If the issue affects critical systems or services, such as Azure access or terminal connections.
